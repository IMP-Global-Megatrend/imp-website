# Payload Migration Baseline Runbook

## Why this exists

This project was previously run in dev mode with dynamic schema pushes. In that mode,
autogenerated baseline migrations can include full schema creation SQL and are unsafe
to run directly on existing databases.

## Baseline artifacts

- Baseline anchor migration: `src/migrations/20260313_103000.ts` (no-op)
- Incremental migration after baseline: `src/migrations/20260313_103216.ts`

## Local workflow (already completed)

1. Confirm migration state:
   - `pnpm payload migrate:status`
2. Apply pending migrations:
   - `pnpm payload migrate`
3. Re-check:
   - `pnpm payload migrate:status`

## Staging / Production rollout

Use the same sequence per environment with environment-specific `DATABASE_URL`:

1. Deploy code containing the baseline and incremental migration files.
2. Run:
   - `pnpm payload migrate:status`
3. If baseline is not marked as ran, apply migrations:
   - `pnpm payload migrate`
4. Verify:
   - `pnpm payload migrate:status`
   - app routes load without missing-column errors.

## Rules after baseline

- Do not run giant autogenerated full-schema migrations against existing environments.
- For every schema change:
  1. `pnpm payload migrate:create`
  2. Review generated SQL carefully
  3. Commit migration + `src/migrations/index.ts`
  4. Run `pnpm payload migrate` per environment
